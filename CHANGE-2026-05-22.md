# gplates module redesign ‚Äî 2026-05-22

This document records the structural redesign of `gadopt/gplates/` carried
out on 2026-05-22, prompted by Dale Roberts' review comments on PRs
#515‚Äì#519 and a follow-up Zulip discussion about the awkward way
geotherm classes reached into their paired indicator classes to share a
SeafloorAgeTracker.

## Motivation

The pre-redesign code folded two orthogonal axes of variation into a
single inheritance chain:

  1. **Where the source points come from** ‚Äî ocean tracker
     (forward-only, stateful) + continental rotator, or polygon filter
     + rotator.
  2. **What we compute at target points** ‚Äî a tanh indicator field, or
     a normalised geotherm (erf for ocean, linear for continental).

Concretely, four classes carried the cross-product of these axes:
`LithosphereConnector`, `PolygonConnector`, `LithosphereGeotherm`,
`PolygonGeotherm`. The two `*Geotherm` classes took an existing
`*Connector` in their constructor and copied its attributes by hand to
share the gtrack state. Their docstring warned the user that "the
indicator connector must be updated before the geotherm at each time
step, because the ocean tracker can only evolve forward in time" ‚Äî a
genuine footgun, since the shared resource was hidden inside one of
its consumers rather than owned by an explicit third object.

Dale's structural suggestion (Zulip thread, 2026-05-21) was to treat
the tracker / rotator / filter as a C-style shared resource: own it in
a dedicated object and have both the indicator and the geotherm hold a
reference. After exploring three design options (factory + shared
backend; Source + Output mixins; single LithosphereField with two
view methods), we picked the Source + Output strategy split.

## What changed

### New modules

- **`gadopt/gplates/sources.py`** ‚Äî `Source` ABC plus
  `LithosphereSource` and `PolygonSource`. Each owns its gtrack
  state and exposes a single `prepare(age) -> dict[str, np.ndarray]`
  call that returns the source-point arrays for a target age. The
  base class implements a per-age cache, which is the central
  correctness property of the redesign: two consumers sharing one
  Source observe a single coherent advance of the underlying
  SeafloorAgeTracker per geological age, regardless of which
  consumer is asked first. Configs `LithosphereSourceConfig` and
  `PolygonSourceConfig` live here, split out of the old monolithic
  `LithosphereConfig` / `PolygonConfig`.

- **`gadopt/gplates/outputs.py`** ‚Äî `OutputStrategy` ABC plus
  `TanhOutput`, `GeothermERFOutput`, `GeothermLinearOutput`. Each
  declares a class-level `requires: ClassVar[frozenset[str]]`
  listing the source-property keys it reads. `MeshConfig` (r_outer,
  depth_scale) and `_DEFAULT_MESH_PARAMETERS` also live here, along
  with the standalone geotherm functions `ocean_erf_normalized` and
  `continental_linear`. The geotherm functions lost the `**kwargs`
  catch-all Dale flagged at gplates.py:1124; signatures are now
  explicit.

### Rewritten modules

- **`gadopt/gplates/connectors.py`** ‚Äî `IndicatorConnector` is now a
  concrete (non-abstract) class composing a Source and an
  OutputStrategy. Construction validates `output.requires <=
  source.provides`, so PolygonSource paired with GeothermERFOutput
  (which needs the `age` key polygons don't carry) fails up front
  with a clear error. The connector owns the kNN `_interpolate`
  step, the result cache keyed by `(age, coords_hash)`, the MPI
  cache-allreduce, and `gc_collect_frequency`. `InterpolationConfig`
  stays here. Dropped: `IndicatorConfigBase`, the `from_dict` /
  `with_overrides` config-routing magic (Dale :135/:160), the
  `_init_checked` lazy attribute audit (Dale :223), the custom dict
  bcast helper (Dale :249/:344 ‚Äî replaced by plain
  `comm.bcast(d, root=0)`).

- **`gadopt/gplates/gplates.py`** ‚Äî `LithosphereConnector`,
  `LithosphereGeotherm`, `PolygonConnector`, `PolygonGeotherm`
  deleted (~870 lines). `LithosphereConfig` and `PolygonConfig`
  removed (replaced by the source configs in `sources.py` plus
  `MeshConfig` in `outputs.py`). The two top-level geotherm
  functions moved to `outputs.py`. Added four factory functions:
  `lithosphere_indicator`, `lithosphere_geotherm`,
  `polygon_indicator`, `polygon_geotherm`. Each takes either a
  pre-built `source=` (for the shared-source case) or the three raw
  constructor args, and returns an `IndicatorConnector`.

- **`gadopt/gplates/gplates.py::GplatesScalarFunction`** ‚Äî applied
  the remaining Dale comments. Vector / tensor function spaces now
  raise a clear `TypeError` at construction (Dale :720). The
  redundant firedrake import dropped (Dale :5). The `r`-as-`self`
  pattern in `_ad_mul` / `_ad_add` (Dale :265, :750) renamed to
  `result`. External API unchanged.

- **`gadopt/gplates/__init__.py`** ‚Äî replaced the bare `from
  .gplates import *` with explicit re-exports so the package's
  public surface is documented by the file itself.

## New tests

Four files under `tests/unit/`. Total ~32 test functions, ~900 LoC.

- **`test_outputs.py`** (31 tests, no fixtures needed) ‚Äî pure-math
  tests of `TanhOutput`, `GeothermERFOutput`, `GeothermLinearOutput`,
  `MeshConfig` validation, `ocean_erf_normalized`, and
  `continental_linear`. Carried over from the old `TestGeothermFunctions`
  and `TestIndicatorComputation` blocks with the `age_myr=None` and
  `**kwargs` branches removed.

- **`test_sources.py`** (20 tests) ‚Äî config validation (no
  reconstruction needed); the `provides` set contract;
  pickled-regression on the dict returned by `prepare(age)` against
  `tests/unit/data/test_lithosphere_source.pkl` and
  `test_polygon_source.pkl`; per-age cache identity
  (`source.prepare(age) is source.prepare(age)`); single-step
  guarantee ‚Äî two connectors sharing one `LithosphereSource` only
  trigger one `_step_ocean_to` call per age (verified via
  `unittest.mock.patch.object` spy).

- **`test_connectors.py`** (13 tests) ‚Äî validation contracts
  (PolygonSource + GeothermERFOutput raises; LithosphereSource +
  GeothermLinearOutput allowed); `GplatesScalarFunction` rejects
  vector / tensor spaces; lockstep regression on all four factory
  pairings against `tests/unit/data/test_connectors.pkl`; shared-
  source consistency (two indicator connectors sharing a source
  produce bit-identical DoF arrays).

- **`test_gplates_adjoint.py`** ‚Äî `MockIndicatorConnector` updated
  to match the new ABC contract (bypasses the
  Source-and-Output-required `__init__`, sets `self.source = mock`
  so that delegated `ndtime2age` / `delta_t` resolve correctly). 16
  tests pass.

- **`test_gplates.py`** ‚Äî age-validation tests retargeted at the
  new factories (`lithosphere_indicator(...)` and
  `polygon_indicator(...)`); semantics unchanged. The old
  `test_lithosphere_connector.py` (1768 lines, 122 methods) and
  `test_polygon_connector_smoothness.py` (436 lines) deleted ‚Äî
  about half their content tested gone-features
  (`to_dict`/`from_dict`/`with_overrides`, the `*_is_indicator_connector`
  inheritance assertions); the remainder is now covered by the four
  new files.

### Regression fixture generation

`tests/unit/data/generate_expected_sources.py` and
`tests/unit/data/generate_expected_connectors.py` produce the
pickled references on demand. They were run once on 2026-05-22
against the Muller 2022 SE v1.2 reconstruction and committed; do
not re-run unless the underlying algorithm intentionally changes.
Tolerances are `rtol=1e-3` for reduced quantities and ¬±1% for
ocean-tracker seed counts (which drift by a handful of points
between runs under floating-point noise).

## Demo

`demos/mantle_convection/gplates_fields/gplates_fields.py` rewritten
to showcase the explicit `LithosphereSource` + two
`IndicatorConnector` triplet idiom for the paired lithosphere
indicator + geotherm fields, and the same pattern for the
continental polygon-bounded pair. The solo crust and craton
indicators use the `polygon_indicator(...)` factory since they
have no paired geotherm. The shared-source intent is now visible
at the call site ‚Äî a reader can see `lith_source` constructed
once and both `I_lith` and `T_erf` holding references to it,
rather than each carrying an invisible duplicate tracker.

`expected.pkl` regenerated against the new code path. The
integrals shifted very slightly (within the existing test's
`rtol=1e-3`) ‚Äî the algorithm is unchanged, so the drift is pure
floating-point reordering inside the source-dict construction.

The smaller `demos/mantle_convection/gplates_lithosphere/`
notebook is **not** updated in this PR. It's a separate, shorter
demo (no geotherm; no shared source) with its own regression
fixture. Updating it would widen the blast radius without adding
clarity to the Source+IndicatorConnector story, which is best
told in `gplates_fields.py`. Tracked as a follow-up.

## Open follow-ups

1. **`gplates_lithosphere/` notebook** ‚Äî update `gplates_lithosphere.ipynb`
   to the new API. Single-source case, no paired geotherm; should be
   a straightforward search-and-replace from `LithosphereConnector(...)`
   to `lithosphere_indicator(...)`.

2. **MPI parallel unit test** ‚Äî `tests/unit/` doesn't use the
   `@pytest.mark.parallel(nprocs=N)` convention; parallel coverage of
   the indicator path currently comes from the demo regression (which
   runs at `cores = 4` via `meta.py`). If the codebase ever introduces
   a parallel-unit-test convention, port a copy of
   `test_connectors.py::TestConnectorRegression` to it.

3. **Logger formatting** ‚Äî `connectors.py:495` (Dale's :495). Would
   need a refactor of the `log()` helper in `gadopt/utility.py` to
   support deferred formatting, which is out of scope for this PR.

4. **PR-chain rebasing** ‚Äî all changes currently sit on the
   integration branch `sghelichkhani/gtrack`. Before pushing for
   review, the slices need to cascade back to their respective PR
   branches:
   - `#515` (gtrack-01-infra-fixes) ‚Äî unaffected; merges independently.
   - `#516` (gtrack-02-indicator-connector-abc) ‚Äî gets the new
     `Source` / `OutputStrategy` ABCs, the rewritten
     `IndicatorConnector`, `MeshConfig`, and the `GplatesScalarFunction`
     cleanup. Rebase via cherry-pick.
   - `#517` (gtrack-03-lithosphere-connector) ‚Äî gets `LithosphereSource`,
     `LithosphereSourceConfig`, `TanhOutput`, `GeothermERFOutput`, and
     the two lithosphere factories. Old `LithosphereConnector` /
     `LithosphereGeotherm` deleted in this PR.
   - `#518` (gtrack-04-polygon-connector) ‚Äî gets `PolygonSource`,
     `PolygonSourceConfig`, `GeothermLinearOutput`, and the two polygon
     factories. Old `PolygonConnector` / `PolygonGeotherm` deleted here.
   - `#519` (gtrack-05-gplates-fields-demo) ‚Äî gets the
     `gplates_fields.py` rewrite and the fresh `expected.pkl`.

5. **`config_extra` ergonomics** ‚Äî the old API let users override one
   parameter via `config_extra={"k_neighbors": 20}`. That convenience
   is gone (Dale's :135/:160 foot-gun was the cost of having it). The
   factories now ask for explicit `interpolation=InterpolationConfig(
   k_neighbors=20)`. If user feedback suggests this is too verbose for
   common one-parameter overrides, a future PR could add a tiny
   `with_overrides`-style helper that doesn't fall into the same
   shallow-copy trap (e.g. by using `dataclasses.replace`).

6. **`TanhOutput.default_thickness_km` ergonomics** ‚Äî the polygon
   factory passes `default_thickness_km=0.0` explicitly; a power
   user constructing the triple manually has to remember this. The
   plan agent flagged this; we kept the explicit form for now so the
   semantics are visible. If multiple users trip over it, consider
   introducing a `PolygonTanhOutput` thin alias with the right
   default.

7. **`LithosphereSource.default_continental_age_myr` vs.
   `GeothermERFOutput.too_far_age_myr`** ‚Äî two distinct things with
   names that almost collide. Renamed `too_far_age_myr` during the
   review pass. Keep an eye on whether documentation or call sites
   confuse them.
